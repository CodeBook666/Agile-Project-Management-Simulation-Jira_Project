Agile-Project-Management-Simulation-Jira_Project

A complete Agile project management simulation for the Litro Gas Ordering System, including user stories, epics, data dictionaries, and wireframes. This repository demonstrates practical skills in requirements gathering, Agile planning, and project documentation.

The Litro Gas Ordering System is a mobile and web-based application that allows customers to order gas cylinders online and manage deliveries efficiently. This repository contains all planning and documentation artifacts created in a simulated Agile environment using Jira. It demonstrates:

- Agile project planning and backlog management
- Structuring epics and user stories for development
- Designing wireframes for user experience
- Documenting data and business rules through a data dictionary
- Artifacts Included:
  
- User Stories: Detailed user stories for both customers and admin/staff
  Examples:
    - As a customer, I want to order a gas cylinder online so that I don’t need to visit the shop.
    - As a staff member, I want to track delivery status so that I can manage customer orders efficiently.
      
- Epics: High-level features broken into multiple user stories
  Examples:
    - Online Ordering System
    - Delivery Management
    - Payment Integration
    - Inventory & Stock Management
      
- Data Dictionary: Defines all entities, attributes, and relationships in the system
  Example tables:
    - Customers – personal details, contact info
    - Orders – order ID, status, payment info
    - Cylinders – stock info, availability
    - DeliveryStaff – staff ID, assigned routes
      
- Wireframes: Visual mockups for the app and admin dashboard
  Includes:
    - Customer app screens: home, order, checkout, profile
    - Staff dashboard screens: order management, delivery tracking, reports
      
- Technologies / Tools Used
    - Jira – for Agile backlog management
    - Figma – for wireframe design
    - Microsoft Excel – for data dictionary and tables
